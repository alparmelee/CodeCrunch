Logic & Proofs

Propositional Logic & Truth Tables ▼

Logical Operators

  • Negation (¬): "not P"
  • Conjunction (∧): "P and Q"
  • Disjunction (∨): "P or Q"
  • Conditional (→): "if P then Q"
  • Biconditional (↔): "P if and only if Q"

Truth Tables

Essential for determining validity of logical statements and arguments.

P Q P → Q P ∧ Q P ∨ Q
T T T T T
T F F F T
F T T F T

Key insight: P → Q is only false when P is true and Q is false.
Conditional Statements ▼

Forms of Conditionals

  • Original: P → Q
  • Converse: Q → P
  • Contrapositive: ¬Q → ¬P (logically equivalent to original)
  • Negation: ¬(P → Q) ≡ P ∧ ¬Q

Example: For "If it rains, then the ground is wet."

Converse: "If the ground is wet, then it rains."

Contrapositive: "If the ground is not wet, then it does not rain."

Quantifiers & Proof Techniques ▼

Quantifiers

  • Universal (∀x): "For all x"
  • Existential (∃x): "There exists an x"
  • ¬(∀xP(x)) ≡ ∃x¬P(x)
  • ¬(∃xP(x)) ≡ ∀x¬P(x)

Proof Methods

  • Direct: Assume P, prove Q follows
  • Contrapositive: Prove ¬Q → ¬P instead of P → Q
  • Contradiction: Assume P and ¬Q, derive contradiction

Graph Theory

Basic Concepts & Trees ▼

Graph Basics

  • Degree: Number of edges at a vertex
  • Handshaking Theorem: Sum of all degrees = 2 × |E|

Trees

  • Connected graph with no cycles
  • |E| = |V| - 1
  • Exactly one path between any two vertices
Euler & Hamilton Paths ▼

Euler Paths/Circuits

  • Euler Path: Uses each edge exactly once
  • Euler Circuit: Euler path returning to start
  • Condition for Euler Circuit: All vertices have even degree
  • Condition for Euler Path: Exactly 0 or 2 vertices have odd degree

Hamilton Paths/Circuits

  • Hamilton Path: Visits each vertex exactly once
  • Hamilton Circuit: Hamilton path returning to start
  • No simple degree-based characterization like Euler paths

Counting & Sets

Set Operations & Counting Principles ▼

Set Operations

  • Union: A ∪ B = {x | x ∈ A or x ∈ B}
  • Intersection: A ∩ B = {x | x ∈ A and x ∈ B}
  • Difference: A \ B = {x | x ∈ A and x ∉ B}
  • Complement: A' = {x ∈ U | x ∉ A}

Counting Rules

  • Sum Rule: |A ∪ B| = |A| + |B| - |A ∩ B|
  • Product Rule: If task A: m ways, task B: n ways, then sequence A→B: m×n ways
  • Permutations: P(n,r) = n!/(n-r)!
  • Combinations: C(n,r) = n!/[r!(n-r)!]

Functions

Types of Functions ▼

Function Properties

  • One-to-one (Injective): Each element in the codomain is mapped to by at most one element from the domain
  • Onto (Surjective): Every element in the codomain is mapped to by at least one element from the domain
  • Bijective: Both one-to-one and onto

Testing one-to-one: If f(a) = f(b), then a = b

Testing onto: For every y in codomain, there exists x in domain where f(x) = y

Piecewise Functions ▼

Definition

Functions defined by different expressions for different parts of the domain.

Example: f(x) = { x² if x ≥ 0, -x if x < 0 }

Key Points

  • Verify continuity at boundary points
  • Check domain and range carefully
  • Ensure complete coverage of intended domain
Sequences & Recurrence Relations ▼

Sequence Types

  • Arithmetic: a_n = a_1 + (n-1)d
  • Geometric: a_n = a_1 × r^(n-1)

Common Recurrence Relations

  • Fibonacci: F_n = F_(n-1) + F_(n-2)
  • Tower of Hanoi: T_n = 2T_(n-1) + 1

Mathematical Induction

Induction Principles ▼

Principle of Mathematical Induction

To prove P(n) for all n ≥ n₀:

  1. Base case: Prove P(n₀) is true
  2. Inductive step: Prove P(k) → P(k+1) for all k ≥ n₀

Strong Induction

To prove P(n) for all n ≥ n₀:

  1. Base case: Prove P(n₀) is true
  2. Inductive step: Prove [P(n₀) ∧ P(n₀+1) ∧ ... ∧ P(k)] → P(k+1) for all k ≥ n₀

Example: Prove 1 + 2 + ... + n = n(n+1)/2

Base case: n = 1: 1 = 1(1+1)/2 = 1 ✓

Inductive step: Assume P(k): 1 + 2 + ... + k = k(k+1)/2

Prove P(k+1): 1 + 2 + ... + k + (k+1) = (k+1)(k+2)/2

[1 + 2 + ... + k] + (k+1) = k(k+1)/2 + (k+1) = (k+1)(k/2 + 1) = (k+1)(k+2)/2 ✓

Common Induction Applications ▼
  • Sum formulas (arithmetic, geometric series)
  • Divisibility proofs
  • Algorithm correctness
  • Inequality proofs
  • Properties of recursively defined sequences

Strategy tip: For divisibility proofs, use factoring or congruence modulo.

Strategy tip: For inequality proofs, consider algebraic manipulations of the induction hypothesis.
